Các bước thiết lập và "đăng nhập" tài khoản Git

1. Kiểm tra Git
   - Chạy: git --version
   - Nếu thấy phiên bản (vd: git version 2.34.1), đã sẵn sàng.

2. Cấu hình thông tin
   - Chạy:
     git config --global user.name "Tên của bạn"
     git config --global user.email "email@example.com"
   - Kiểm tra: git config --list
   - (Email phải trùng với GitHub)

3. Tạo SSH Key (Khuyến nghị)
   - Tạo key:
     ssh-keygen -t ed25519 -C "vuduyhien1992@gmail.com"
     (hoặc ssh-keygen -t rsa -b 4096 -C "vuduyhien1992@gmail.com" nếu không hỗ trợ ed25519)
   - Nhấn Enter để dùng mặc định, bỏ qua passphrase nếu muốn.
   - Khởi động SSH Agent: eval "$(ssh-agent -s)"
   - Thêm key:
     ssh-add ~/.ssh/id_ed25519 (hoặc ~/.ssh/id_rsa)
   - Sao chép key: cat ~/.ssh/id_ed25519.pub (hoặc id_rsa.pub)
   - Thêm vào GitHub: Settings > SSH and GPG keys > New SSH key > Dán key > Add.
   - Kiểm tra: ssh -T git@github.com (Thấy "Hi username!" là thành công)

4. Kết nối Repository
   - Clone: git clone git@github.com:username/repository.git
   - Thêm remote: git remote add origin git@github.com:username/repository.git

5. Push thử
   - Vào thư mục: cd repository
   - Tạo file: echo "Test" > test.txt
   - Commit: git add test.txt && git commit -m "Test commit"
   - Push: git push origin main

6. (Tùy chọn) Dùng PAT thay SSH
   - Tạo PAT: Settings > Developer settings > Personal access tokens > Generate new token
   - Dùng HTTPS: Username = Tên GitHub, Password = PAT

Kết quả:
   - Đã kết nối Git với GitHub qua SSH hoặc PAT.
   - Có thể clone/push/pull dễ dàng.

Xử lý lỗi:
   - Lỗi SSH: ssh -vT git@github.com
   - Permission denied: Kiểm tra lại key trên GitHub.

Fix bằng câu lệnh khi chạy kiểm tra mà không thấy Hi username!
   - ssh-keyscan github.com >> ~/.ssh/known_hosts
